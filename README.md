> **حالة المشروع:** 🚀 قيد التطوير النشط

## 📖 نظرة عامة على المشروع

نظام محاسبي وجرد يومي مدمج لتحويل العمليات اليومية الورقية إلى منظومة رقمية دقيقة وآمنة. يوفر إدارة شاملة للموردين (الرعاوي)، الزبائن (المعايشة)، الفواتير، الديون، والمستحقات المالية.

### 🎯 الهدف الأساسي
- تحويل دفتر البيع اليومي الورقي إلى نظام رقمي آمن وموثوق
- ضمان عدم ضياع الديون أو حقوق الموردين والزبائن
- توفير الوقت والجهد في تسجيل وحساب العمليات
- إعداد تقارير دقيقة وفورية للمبيعات والأرباح

## 📋 المتطلبات الرئيسية

### FR-1: إدارة المستخدمين والصلاحيات ✅
- نظام تسجيل دخول آمن
- ثلاثة أدوار: مدير (المحرج)، كاشر، محاسب
- نظام تدقيق شامل (Audit Log)
- تحكم بالأجهزة المرخصة

### FR-2: إدارة الرعاوي (الموردين)
- بيانات تفصيلية للرعوي
- كشف حساب مجمع يومي
- إدارة السلف والمسحوبات
- تتبع المستحقات

### FR-3: إدارة المعايشة (الزبائن)
- ملف شامل للزبون مع بيانات شخصية وضمانات
- حد ائتماني قابل للتحكم
- فواتير مجمعة يومية

### FR-4: الفواتير الذكية
- منع إنشاء فواتير مكررة لنفس الزبون في نفس اليوم
- دمج تلقائي للمبيعات المتكررة
- تسعير يدوي 100%

### FR-5: إدارة الديون والسداد
- خوارزمية FIFO للسداد الذكي
- ترحيل الكسور المالية
- تجميد المطالبات للعملاء المتعثرين

### FR-6: حسابات العمولات والضرائب
- سعاية 7% من المبيعات
- ضريبة يومية ثابتة 500 ريال
- إدارة المصروفات

### FR-7: إدارة الصندوق
- تتبع رصيد الصندوق
- جرد الدرج (عجز/فائض)
- عمليات الإيداع والسحب

### FR-8: التقارير والتكاملات
- لوحة تحكم (Dashboard)
- تقارير أعمار الديون
- تكامل WhatsApp للتذكيرات
- تصدير PDF

## 🛠️ التقنيات المستخدمة

- **Frontend:** React + TypeScript + Tailwind CSS
- **Backend:** Node.js + Express (أو Supabase)
- **Database:** SQLite (محلي) + Cloud Sync (Firebase/Supabase)
- **Mobile:** React Native / Flutter
- **State Management:** Redux أو Context API
- **Testing:** Jest + React Testing Library
- **Security:** Encryption, JWT, Device ID verification
